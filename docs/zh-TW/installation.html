安裝

title: 安裝

必要條件：NPM（node 內建）或 Yarn。

如果要全域安裝 BuckleScript，請執行：

npm install -g bs-platform

安裝後會有幾個可以全域執行的指令，稍候介紹。

npm install --save-dev bs-platform

Windows 請注意，如果同時使用 BuckleScript 及 Merlin，請參考這個分享給 Reason 的指南。

可供全域執行的指令有：

  • bsb，建置系統。
  • bsc，原生編譯器。通常不會直接使用。
  • bsrefmt，內建 Reason 剖析 & 印出器.

替代方案（進階用戶）

透過 NPM/Yarn，使用程式碼安裝

必要條件：npm 或 yarn，外加標準 C 工具鏈。

git clone https://github.com/bucklescript/bucklescript
cd bucklescript
npm install

不透過 NPM/Yarn，使用程式碼安裝

必要條件：標準 C 工具鏈（gcc、make）。

首先，建置 OCaml 編譯器：

git clone https://github.com/bucklescript/bucklescript
cd bucklescript/vendor/ocaml
./configure -prefix `pwd` # 填入你所希望的安裝位置
make world.opt
make install

編譯器將安裝於 $(pwd)/bin。 若要暫時地使用，檢查 ocamlc.opt 和 ocamlopt.opt 使否存在 $(pwd)/bin，並暫時地加入路徑於 $(PATH)（例如：PATH=$(pwd)/bin:$PATH）。

接著，建置 BuckleScript 本身：

cd ../../
make world

最後，位於 jscomp/bin 目錄底下會有個二進制檔案 bsc.exe，可以加入到你的 $PATH。 另外也可以設定環境變數，指定標準函式庫的位置，例如 BSC_LIB=/path/to/jscomp/stdlib，方便使用。

警告： 建置出來的編譯器不能隨意更換目錄位置，請不要隨意搬動，除非你知道在做什麼！
