Installation

title: Installation

Prérequis: Soit avec NPM (fournis avec node) ou Yarn.

Si vous voulez installer BuckleSript globalement :

npm install -g bs-platform

Cela vous donne quelques commandes exposées globalement que vous pouvez exécuter, décrites plus tard. Mais généralement, vous devez installer le projet localement:

npm install --save-dev bs-platform

A noter pour Windows, si vous utilisez BuckleScript en relation avec un outil comme Merlin, merci de suivre ce guide que nous partageons avec Reason.

Les commandes exposées sont:

  • bsb, the build system.
  • bsc, the raw compiler. Usually not used directly.
  • bsrefmt, the included Reason parser & printer.

Alternatives (Utilisateurs Avancés)

Installation depuis les sources, avec NPM/Yarn

Prérequis: soit avec NPM ou Yarn, ainsi que les outils de compilation standard de C.

git clone https://github.com/bucklescript/bucklescript
cd bucklescript
npm install

Installation depuis les sources, Sans NPM/Yarn

Prérequis: les outils standard de compilation pour C (gcc, make).

Tout d'abord, construire le compilateur OCaml:

git clone https://github.com/bucklescript/bucklescript
cd bucklescript/vendor/ocaml
./configure -prefix `pwd` # put your preferred directory
make world.opt
make install

Le compilateur patché est installé localement dans $(pwd)"/ bin". Pour commencer à l'utiliser temporairement, vérifier si ocamlc.opt et ocamlopt.opt existent dans $(pwd)/bin, et ajouter le temporairement à votre $(PATH) (e.g. PATH=$(pwd)/bin:$PATH).

Ensuite, construire BuckleScript lui-même :

cd ../../
make world

À la fin, vous devriez avoir un fichier binaire appelé bsc.exe sous le répertoire jscomp/bin, que vous pourrez ajouter à votre $PATH. Vous pouvez également définir une variable d’environnement pointant vers la bibliothèque standard, par exemple BSC_LIB = / chemin/de/jscomp/stdlib, pour faciliter l’utilisation.

Attention: le compilateur n'est pas réadressable out of the box, ne pas le déplacer sauf si vous savez ce que vous faites !
