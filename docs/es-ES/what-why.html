Qué y para qué

¿Que es uckeScript?

BuckeScript no es un nuevo lenguaje. Simplemente tomaOCaml,un lenguaje rapido,pragmatico y mecanografiado, y hace compilacion para limpiar, legible y codigo de rendimiento JavaScript. Esto permite a los usuarios escribir en una lengua solida mientras que utilizando las herramientas de JavaScript existentes, familiar (npm/hilado, empaquetadoras, minifiers, devtools, etc.) y acceder al gran ecosistema de ambas Javascript y OCaml.

Gracias a su gran interoperatividad con JS, BuckeScript puede ser visto como un lenguaje de tipo "gradualmente" donde se llega a adoptar progresivamente en su codigo base archivo por archivo, manteniendo todas las ventajas de un tipo confiable y 100% sistema de sonido, mas extra caracteristicas de lenguaje.

BuckleScript tiene soporte incorporado para razon, un proyecto socio nuestro, para brindar una experiencia aun mas familiar a los desarrolladores de JS.

¿ Por que compilar JavaScript?

Ecosistema y comunidad de JavaScript son muy vibrante y activo. La lengua en si mismo es incomparable en terminos de adopcion& distribucion. Compilacion de JavaScript es una opcion, por tanto pragmatica ( y a veces la unica opcion).

Adicionalmente, el VMs de JavaScript tambien recibio muchos años de ingenieria de puesta a punto y llegar mas rapido todo el tiempo.

¿ Por que BuckleScript?

"¿Por que no usar nada pero JavaScript entonces?"

A pesar de sus claras ventajas, JavaScript presenta algunos inconvenientes. Aqui es como BuckeScript resuelve.

Fiabilidad

El sistema de tipo BuckleScript solo ofrece tres beneficios principales para hacer sus programas mas confiable:

  • Sistema de tipo sonido. "Sonido" aqui significa los tiposgarantizarque son lo que son, no solo el 90% del tiempo. Una vez que compila un proyecto de BuckeScript, no hay ningun tipo de error de tiempo de ejecucion. /*
  • Inferencia de tipo fuerte. Casi la totalidad de la lengua puede ser inferida. Usted no tiene que escribir tediosamente los tipos manualmente. Sientase libre de escribir algunas partes para la legibilidad!
  • Caracteristicas de tipo expresivo. Con las caracteristicas del bien-pensamiento-hacia fuera comovariantes,modulosincluso un opt-ensistema de objetos,los tipos guiaran su proceso de iteraccion y no bloqueara de expresar lo que usted necesita.

La combinacion de estos tres reflejos, junto con otras caracteristicas del sistema tipo, abre nuevas posibilidades de flujo de trabajo y es algo que no ofrecen soluciones existentes de tipeo graduales para JavaScript.

*las excepciones son unos ganchos de interoperatibilidad deliberada, opcional.

Eliminacion de codigo muerto de alta calidad

El ecosistema JavaScript es muy confiado de las dependencias y a menudo muchos de ellos. Envio del producto final inebitablemente arrastra en una gran cantidad de codigo, que muchas de las cuales el proyecto no utiliza realmente. Estas regiones de codigo muerto impactan cargando, analisis y velocidad de interpretacion. BuckleScript proporciona eliminacion de codigo muerto de gran alcance en todos los niveles:

  • Eliminacion de codigo de nivel de modulo y funcion es facilitada por el sistema de tipo bien diseñada y analisis de pureza.
  • A nivel global, BuckleScript genera codigos que son naturalmente amigables para la eliminacion de codigo muerto hecho ya que integra herramientas tales comoRollupyClosure Compiler,despues de su pasada eliminacion sofisticada.
  • Lo mismo aplica para el poquito tiempo de ejecucion de BuckleScript ( que se escribe en BuckleScript a si mismo).

Optimizaciones de tiempo de compilacion

JavaScript es un lenguaje dinamico: tiene un rendimiento cuando el motor del navegador optimiza el codigo en tiempo de ejecucion. Mientras que algunos motores de sortear el problema hasta cierto punto porcaching,esto no es siempre previsible ni disponible. La falta de un sistema de tipo fuerte, incorporado tambien limita la cantidad de posibles optimizaciones. BuckleScript por otra parte, es capaz de proporcionar muchas optimizaciones en tiempo de compilacion, mucho antes de que el codigo es enviado al navegador. Un amplio refran para escribir codigo JavaScript rapido es escribir como si hay un tipo de sistema (para activar heuristica de buena optimizacion de los mejores JS); BuckleScript le da a usted uno verdadero y genera codigo que es amigable con las optimizaciones de forma predeterminada.

! Funciona en Nativo tambien!

BuckleScript es realmente OCaml( aparte de la interoperatividad de javaScript). OCaml puede compilar directamente en codigo maquina (aka rapido, montaje de barebone)*. Cuando usted no esta eligiendo como objetivo JavaScript, se puede reusar el mismo codigo y servidores de destino, escritorios, sistemas operativos moviles, etc. Esto desbloquea incluso mejor rendimiento. Por ejemplo, la aplicacion tradicional de node.Js comienza en100ms,mientras OCaml compilada a codigo maquina comienza en alrededor de 2ms.

*De hecho, incluso funciona enmicrocontroladorescon solo unos pocos KBsde memoria.

¿Como es BucleScript diferente?

Hoy en dia, hay muchos idiomas de compilacion a JavaScript. BucleScript se distingue de las alternativas en algunos aspectos crucuiales.

Ciclo de Iteración Rápida

Tiempo de construccion BuckleScript esuno o dos ordenes de magnitudmas rapido que las alternativas. En su modo de observador, el sistema de construccion generalmente termina antes de cambiar la pantalla del editor a la pestaña terminal ( dos digitos de milisegundos). Un ciclo de iteraccion rapida reduce la necesidad de mantener el estado mental alrededor mas largo; Esto en los giros permite estar en el flujo mas tiempo y mas a menudo.

Salida legible & y gran interoperatibilidad

Ilegible codigo JavaScript generado de otros idiomas compiladas a Js hace que puede ser, practicamente hablando:

  • Dificil de depurar
  • Dificil de aprender de
  • Duro al perfil del desempeño
  • Dificil de integrar con el vigente codigo Js de escritos a mano

La salida de BuckleScript es muy elegible. Esto es especialmente importante mientras se aprende, donde los usuarios pueden aprender como el codigo compilado y auditoria de errores.

Esta caracteristica, combinada con todas las funciones JS interoperatibidad sistema, permite a BuckleScript codigo para insertar en un JavaScript existente codigo casi inadvertido.

Salida JS pequeñitos

Un programaHola mundoBuckleScript genera codigo Js de20 bytesJs codigo. ademas, la biblioteca estandar de piezas que usted requiere en son solo incluidas cuando sea nesesario.

Preservacion de la estructura del codigo

BuckleScript mapas de un archivo de OCaml/razon a un archivo de JavaScript. Esto facilita la integracion de las herramientas existentes como empaquetadoras y corredores de prueba. Usted incluso puede comenzar a escribir un solo archivo sin mucho cambio la construccion de su configuracion. La estructura del codigo de cada archivo es aproximadamente preservado, tambien.
