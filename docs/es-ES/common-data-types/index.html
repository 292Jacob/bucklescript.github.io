Tipos de datos comunes

Tipos de Datos Compartidos

Los primitivos de BuckleScript como string, float, array y algunos otros tienen una propiedad algo interesante: ¡se compilan a exactamente el mismo JavaScript! Gracias a esto, no hay casi ninguna API que no lea estos tipos de datos.

Esto significa que si recibes por ejemplo una cadena del lado de JS, puedes usarla sin conversión en el lado de BS, y viceversa. En otras palabras, cadenas y otros son "representaciones públicas garantizadas".

P.S. si necesitas un resumen del lenguaje de los primitivos, ve los documentos de Reason.

BuckleScript usa la misma librería estándar de OCaml; ve los documentos aquí (en la sintaxis de Reason). Additionally, we provide access to all the familiar JS primitives here. Puedes mezclar y emparejar estos dos.

Cadena

Inmutable en ambos lados, como es de esperar. API de la Cadena de BuckleScript. API de la Cadena de JS.

Soporte de Unicode

La cadena de OCaml es una secuencia de byte inmutable. Si el usuario escribe algún unicode:

Js.log "你好"

Js.log("你好")

Se compilará al siguiente JS:

console.log("\xe4\xbd\xa0\xe5\xa5\xbd");

Lo que te da una salida críptica en la consola. Para rectifica esto, BuckleScript expone un anotacion js especial a la sintaxis de citación de cadenas por defecto construida en el lenguaje. Úsala como esto:

Js.log {js|你好，
世界|js}

Js.log({js|你好，
世界|js})

Esto correctamente dará:

console.log("你好，\n世界");

Interpolación

Por conveniencia, también queremos exponer otra anotación especial para las etiquetas citadas, j, que soporta el equivalente a la interpolación de una cadena JS:

let world = {j|世界|j}
let helloWorld = {j|你好，$world|j}

let world = {j|世界|j};
let helloWorld = {j|你好，$world|j};

Puedes rodear la variable en paréntesis también: {j|你好，$(world)|j}.

Flotantes

Los flotantes en BuckleScript son números en JS, y viceversa. La librería estándar de OCaml no viene con un módulo de Flotantes. La API de Flotantes JS está aquí.

Int

¡Los Ints son 32-bits! Se cuidadoso, potencialmente los puedes tratar como números JS y viceversa, pero si el número es largo, es mejor que trates los números JS como flotantes. Por ejemplo, nosotros enlazamos a Js.Date usando floats. La API de Int de JS está aquí.

Arreglo

Los arreglos idiomáticos de OCaml supuestamente tienen que tener un tamaño fijo. Esta restricción es liberada en el tamaño de BuckleScript. Puedes cambiar su longitud usando la API de Arreglos de JS usual. La API propia de BuckleScript para los Arreglos está aquí.

Tupla

Las Tuplas OCaml son compiladas a Arreglos JS. Convenientes cuando estas haciendo un interlop con un arreglo JS que contiene valores heterogéneos, pero que pasan a tener una longitud fija. ¡Modélalo como una tupla en el lado de BS!

Bool

Since BuckleScript 3, OCaml/Reason bool now compile to JS boolean.

Tipos de Datos No Compartidos

Record, variant (including option and list), object and others can be exported as well, but you should not rely on their internal representation on the JS side. Aka, don't grab a BS list and start manipulating its structure on the JS side.

However, for record and variant, we provide generation of converters and accessors. Once you convert e.g. a record to a JS object, you can naturally use them on the JS side.

For list, use Array.of_list and Array.to_list in the Array module. option will be treated later on.

Design Decisions

As to why we don't compile list to JS array or vice-versa, it's because OCaml array and JS array share similar characteristics: mutable, similar read/write performance, etc. List, on the other hand, is immutable and has different access perf characteristics.

The same justification applies for records. OCaml records are fixed, nominally typed, and in general doesn't work well with JS objects. We do provide excellent facilities to bind to JS objects in the object section.
