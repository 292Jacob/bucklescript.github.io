Introduccion a externo

Muchas partes del sistema de interop usa un concepto llamado external, así que lo presentaremos especialmente aquí.

external es una palabra clave para declarar un valor en BuckleScript/OCaml/Reason:

external myCFunction : int -> string = "theCFunctionName"

Sintaxis de Reason:

external myCFunction : int => string = "theCFunctionName";

Es como un let, excepto que el cuerpo de un external es, como se vio anteriormente, una cadena. Esa cadena usualmente tiene significados específicos dependiendo del contexto. Para OCaml nativo, usualmente apunta a una función C de ese nombre. Para BuckleScript, estos externals son usualmente decorados con un cierto atributo [@bs.blabla].

Una vez declarado, puedes usar un external como un valor normal.

BuckleScript externals are turned into the expected JS values, inlined into their callers during compilation, and completely erased afterward. You won't see them in the JS output. It's as if you wrote the generated JS code by hand! No performance cost either, naturally.

Note: do also use externals and the [@bs.blabla] attributes in the interface files. Otherwise the inlining won't happen.

External de Identidad Especial

Un external que vale la pena mencionar es el siguiente:

external myShadyConversion : foo -> bar = "%identity"

Sintaxis de Reason:

external myShadyConversion : foo => bar = "%identity";

Este es una escotilla de escape final que no hace nada más que convertir del tipo foo al bar. In the following sections, if you ever fail to write an external, you can fall back to using this one. Pero intenta no hacerlo.
