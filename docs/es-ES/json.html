JSON

Hay distintas maneras de trabajar con JSON en este momento, que unificaremos y puliremos muy pronto.

Conversión Insegura

Esto emula una conversión JSON de JavaScript.

Analizar

Simplemente usa el (último recurso) especial external de identidad:

type data = < name :string  > Js.t
external parseIntoMyData : string -> data = "parse" [@@bs.scope "JSON"][@@bs.val]

let result = parseIntoMyData "{\"name\": \"Luke\"}"

Sintaxis de Reason:

type data = {. "name": string};
[@bs.scope "JSON"][@bs.val] external parseIntoMyData: string => data = "parse";

let result = parseIntoMyData("{\"name\": \"Luke\"}");

Salida:

var result = JSON.parse("{\"name\": \"Luke\"}");

Donde data puede ser de cualquier tipo que tu asumas que es JSON. Como puedes ver, esto compila directo a una llamada JSON.parse. Como un JS regular, esto es conveniente, pero no tiene garantía de que por ejemplo, los datos ester formados correctamente, o siquiera sintácticamente válidos.

Encadenamiento

Como el JSON.stringify es un poco más seguro que JSON.parse, lo hemos proporcionado fuera de la caja en Js.Json. Esto compila a JSON.stringify.

Uso correcto de Js.Json

Técnicamente, la manera correcta de manejar JSON es para analizar recursivamente cada campo, y manejar datos inválidos de manera acorde. Js.Json provee un tipo de bloques de construcción de bajo nivel. Ve el ejemplo en los documentos de la API.

Ayudantes de Alto Nivel

Tenemos un ayudante pseudo-oficial, mantenido por la comunidad, de las librerías de JSON, llamado bs-json. Aquellos interesados en combinadores también pueden revisar JsonCodec. Ambos proveen ayudantes de análisis/impresión más conveniente JSON que aprovechan las secciones anteriores de APIs de bajo nivel.

Proporcionaremos una mejor, y de primera mano, solución para JSON también en el futuro. ¡Atentos!
