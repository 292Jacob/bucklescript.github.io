nuevo

Use bs.newpara simular e.g. new Date():

type t
external createDate : unit -> t = "Date" [@@bs.new]

let date = createDate ()

Sintaxis de la Razón:

type t;
[@bs.new] external createDate : unit => t = "Date";

let date = createDate();

Salida:

var date = new Date();

Usted puede encadenar bs.new and bs.module si el modulo de JS que usted esta importando es una clase:

type t
external book : unit -> t = "Book" [@@bs.new] [@@bs.module]
let bookInstance = book ()

Sintaxis de la Razón:

type t;
[@bs.new] [@bs.module] external book : unit => t = "Book";
let bookInstance = book();

Salida:

var Book = require("Book");
var bookInstance = new Book();

Unir a Clases JS

Las clases JS son realmente objetos JS conectados graciosamente. En general, preferimos usar las características de objetos de secciones previas para enlazar a un objeto JS.

OCaml having classes really helps with modeling JS classes. Just add a [@bs] to a class type to turn them into a Js.t class:

class type _rect = object
  method height : int [@@bs.set]
  method width : int [@@bs.set]
  method draw : unit -> unit
end [@bs]

type rect = _rect Js.t

Sintaxis de Reason:

class type _rect =
  [@bs]
  {
    [@bs.set] pub height: int;
    [@bs.set] pub width: int;
    pub draw: unit => unit
  };

type rect = Js.t(_rect);

Para clases Js.t, métodos con tipos de flecha son tratados como métodos reales (automáticamente anotados con [@bs.meth]) mientras los métodos con tipos sin flecha son tratados como propiedades. Añadir abs.set a esos métodos los harán mutables, lo que permite establecerlos usando #= después. Soltar el atributo bs.set hace al método/propiedad inmutable. Básicamente, como las características en la sección de objeto.
