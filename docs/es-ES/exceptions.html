Excepciones

En el mundo JS, las excepciones podrían ser cualquier dato, mientras que una excepción de OCaml es un formato de datos estructurados y da soporta a la coincidencia de patrones. Capturar una excepción OCaml en el lado JS no funciona como se es debido.

Las excepciones de JS pueden elevarse desde el lado de BuckleScript usando las funciones the JS.Exn.raise*, y puede ser atrapado como una excepción BS de tipo Js.Exn.Error con la excepcion JS como su carga, de tipo Js.Exn.t. La excepción JS cualquiera puede ser manipulado con las funciones de acceso en Js.Exn, o fundido a un tipo mas apropiado.

intente 
  Js.Exn.raiseError "oops!"
with
| Js.Exn.Error e ->
  match Js.Exn.message e with
  | Some message -> Js.log {j|Error: $message|j}
  | None -> Js.log "An unknown error occurred"

intente  (
  Js.Exn.raiseError("oops!")
) {
| Js.Exn.Error(e) =>
  switch (Js.Exn.message(e)) {
  | Some(message) => Js.log({j|Error: $message|j})
  | None => Js.log("An unknown error occurred")
  }
};

Uso

Take promise for example:

exception UnhandledPromise

let handlePromiseFailure = function [@bs.open]
  | Not_found ->
    Js.log "No encontrado";
    Js.Promise.resolve ()

let _ =
 Js.Promise.reject Not_found
 |> Js.Promise.catch (fun error ->
      match handlePromiseFailure error with
      | Some x -> x
      | None -> raise UnhandledPromise
  )

exception UnhandledPromise;

let handlePromiseFailure =
  [@bs.open]
  (
    fun
    | Not_found => {
        Js.log("No encontrado");
        Js.Promise.resolve()
      }
  );

Js.Promise.reject(Not_found)
  |> Js.Promise.catch(
     (error) =>
       switch (handlePromiseFailure(error)) {
       | Some(x) => x
       | None => raise(UnhandledPromise)
       }
   );
