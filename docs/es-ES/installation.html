Instalación

título: Instalación

Prerequisito: ya sea NPM (viene con node) o Yarn.

Si te gustaría instalar BuckleScript globalmente haz lo siguiente:

npm install -g bs-platform

Esto da algunos comandos globales expuestos que puedes ejecutar, descritos después. Pero usualmente, instalarías el proyecto localmente:

npm install --save-dev bs-platform

Ten en cuenta para Windows, si estás usando BuckleScript en conjunto con una herramienta como Merlin, por favor sigue esta guía que compartimos con Reason.

Los comandos que están expuestos son:

  • bsb, el sistema de construcción.
  • bsc, el compilador crudo. Usualmente no se usa directamente.
  • bsrefmt, el analizador & impresor qde Reason que viene incluido.

Alternativas (Usuarios de Poder)

Instalar desde Origen, Mediante NPM/Yarn

Prerequisito: ya sea npm o yarn, además de la herramienta de cadena de C estándar.

git clone https://github.com/bucklescript/bucklescript
cd bucklescript
npm install

Instalar desde Origen, Sin NPM/Yarn

Prerequisito: La herramienta de cadena de C estándar (gcc, make).

Primero, construye el compilador OCaml:

git clone https://github.com/bucklescript/bucklescript
cd bucklescript/vendor/ocaml
./configure -prefix `pwd` # put your preferred directory
make world.opt
make install

El compilador parcheado está instalado localmente en $(pwd)/bin. Para empezar a usarlo temporalmente, verifica si ocamlc.opt y ocamlopt.opt existen en $(pwd)/bin, y temporalmente añade la ubicación a tu $(PATH) (p.e. PATH=$(pwd)/bin:$PATH).

Entonces, construye BuckleScript como tal:

cd ../../jscomp
make world

Al final deberías tener un binario llamado bsc.exe dentro del directorio jscomp/bin, que puedes añadirlo a tu $PATH. Además puedes establecer una variable de entorno señalando a la librería estándar, p.e. BSC_LIB=/path/to/jscomp/stdlib, para facilitar el uso.

Advertencia: el compilador construido no es reubicable fuera de la caja, ¡no lo muevas a menos que sepas qué estás haciendo!
