Enlazar a valores globales

Do you want to use window? ¿O Math? ¿O cualquier cosa que no tengas que primero import o require en JavaScript? This section describes how to access those from BuckleScript.

First, make sure the value you'd like to model doesn't already exist in our provided API. Para una búsqueda rápida de valores, ve el índice de valores.

Ahora, asegúrate que no está ya en https://github.com/reasonml-community o NPM.

Ahora, aquí está como enlazas a un valor JS:

external setTimeout : (unit -> unit) -> int -> float = "setTimeout" [@@bs.val]
external clearTimeout : float -> unit = "clearTimeout" [@@bs.val]

Sintaxis de Reason:

[@bs.val] external setTimeout : (unit => unit, int) => float = "setTimeout";
[@bs.val] external clearTimeout : float => unit = "clearTimeout";

Esto enlaza en el JavaScript métodos setTimeout y el correspondiente clearTimeout. El tipo de anotación de external especifica que setTimeout:

  • takes a function that accepts unit and returns unit (which on the JS side turns into a function that accepts nothing and returns nothing. More on modeling functions later)
  • y un entero que especifica la duración antes de llamar dicha función
  • returns a number that is the timeout's ID. This number might be big, so we're modeling it as a float rather than the 32-bit int

Consejos & Sugerencias

Taquigrafía

When the name you're using on the BS side matches the JS value you're modeling, you can use the empty string shorthand:

external clearTimeout : float -> unit = "" [@@bs.val]

Sintaxis de Reason:

[@bs.val] external clearTimeout : float => unit = "";

Tipo Abstracto

The above still isn't ideal. Ve como setTimeout regresa un float y cleatTimeout acepta uno. ¡No hay garantía de que estás pasando el float creado por setTimeout en clearTimeout! Por todo lo que sabemos, alguien podría pasar un Math.random() en el último.

¡Estamos en un lenguaje con un gran sistema de tipos ahora! Vamos a tomar ventaja de una característica popular para resolver este problema: tipos abstractos.

type timerId
external setTimeout : (unit -> unit) -> int -> timerId = "setTimeout" [@@bs.val]
external clearTimeout : timerId -> unit = "clearTimeout" [@@bs.val]

Sintaxis de Reason:

type timerId;
[@bs.val] external setTimeout : (unit => unit, int) => timerId = "setTimeout";
[@bs.val] external clearTimeout : timerId => unit = "clearTimeout";

Claramente, ¡timerId es un tipo que sólo puede ser creado por setTimeout! Ahora hemos asegurado que a clearTimeout se le va a pasar un ID válido. Ya sea un número bajo el capó ahora es un mero detalle de implementación.

Inspecciona la salida aquí. Es tan limpio como un código JS escrito a mano, excepto que sabes que viene desde un BucleScript correctamente tipeado. If these kind of features are all you use from BuckleScript, you'd already have derived values from them!

Este truco es utilizado frecuentemente para permitir que otros concuerden en que funcionalidades JS enlazar, sin prescribir como hay que enlazarlas. Por ejemplo, la librería BS expone unos pocos tipos abstractos como window. The DOM API is huge and extremely hard to bind to correctly; we don't provide an opinionated way of doing it, so exposing the types and allowing you to use them as input/output types of your externals and have everyone's opinionated wrapper still agreeing on the types is a great middle-ground. You, for example, might only need 3 methods from DOM, and so wrote your own, thin wrappers for them.

Módulos Globales

Si quieres enlazar un valor dentro de un módulo global, p.e. Math.random, atrae un bs.scope a tu bs.val externo:

external random: unit -> float = "random" [@@bs.val][@@bs.scope "Math"]
let someNumber = random ()

Sintaxis de Reason:

[@bs.scope "Math"] [@bs.val] external random : unit => float = "random";
let someNumber = random();

puedes enlazar a un objeto de profundidad arbitraria pasando una tupla a bs.scope:

external length: int = "length" [@@bs.val][@@bs.scope "window", "ancestorOrigins", "ancestorOrigins"]

Sintaxis de Reason:

[@bs.val] [@bs.scope ("window", "ancestorOrigins", "ancestorOrigins")] external length : int = "length";

This binds to window.location.ancestorOrigins.length.
