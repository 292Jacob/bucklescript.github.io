Overview

BuckleScript incorpora su propio sistema de build, Bsb, que aspira a ser rápido, ligero y estándar de facto en la comunidad a la hora de construir projectos.

Bsb provee algunos templates para crear proyectos rápidamente:

bsb -init nombre-del-directorio

Para generar un proyecto Reason:

bsb -init nombre-del-directorio -theme basic-reason

Puedes inspeccionar los archivos en el directorio recién generado. Para ver todos los templates disponibles, ejecuta:

bsb -themes

El archivo de configuración de la build se llama bsconfig.json. Todos los proyectos que usan BuckleScript necesitan uno.

Para compilar un proyecto y sus dependencias, ejecuta:

bsb -make-world

Añade -w para ejecutar el watcher integrado. Cualquier cambio en los archivos hará que la build se regenere. Nota: Los archivos en node_modules no se supervisan ya que se podría exceder el límite de archivos que node.js maneja por defecto.

Para construir únicamente el proyecto, utiliza bsb -make.

Ejecuta bsb -help para ver todas las opciones disponibles.

Limpieza de archivos intermedios

En raras ocasiones, la build puede corromperse, si se da el caso ejecuta:

bsb -clean-world

O bsb -clean para purgar sólo los archivos del proyecto, sin incluir las dependencias.

Soporte de editores

Bsb generates a .merlin file, used by various editor plugins under the hood to power e.g. autocomplete, type hint, diagnosis, etc.

Bonus: Puedes transferir los errores de terminal de bsb a VSCode usando esta configuración.

Consejos y sugerencias

Un problema usual de los sistemas de build tradicionales es que no son fiables a la hora de mover o borrar archivos fuente. La mayoría no borran los archivos intermedios correctamente después de estas acciones*. Bsb, desafortunadamente, no es diferente a estos sistemas. A no ser que actives "suffix": ".bs.js" en bsconfig.json, en cuyo caso bsb podrá rastrear qué archivos intermedios JS corresponden con cada archivo fuente, incluso después de mover o borrar estos archivos.

Decisiones de diseño

* Uno de los sistemas que monitoriza correcta y eficientemente estos cambios es Tup. Puedes ver el (bastante accesible!) artículo aquí. Desafortunadamente, la implementación de Tup utiliza FUSE, el cual no podemos utilizar con garantías en todas las plataformas.
