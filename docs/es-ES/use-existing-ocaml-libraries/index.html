Utilizar las bibliotecas existentes de OCaml

Esta seccion esta reservada para usos avanzados. Generalmente,Usted escogeria una biblioteca de OCaml/razon que ya esta configurada para trabajar con nuestro sistema de construir de alto nivelbsb(aka tienebsconfig.json). Si no, generalmente seria añadir unbsconfig.jsona el. Pero si usted quiere utilizar directamente un paquete de OCaml que no forma parte del flujo de trabajo de MNP, o estan construyendo algunos infra peculiares con BucleScript, siga leyendo!

Esta guia es tambien un pequeño tutorial de como bsbtrabaja bajo el capo.

Construido en apoyo NPM

Construir una biblioteca OCaml como un paquete NPM

Nota: esta seccion podria ser ligeramente rancia. Si alguno de los pasos no funciona, por favor archivarnos un problema! Gracias.

le recomendamos altamenteque usted intente este esfuerzo en una biblioteca de menor dependencia primero. evitaria muchos problemas.

Compilador de BuckleScript,bscamplia las opciones de compilador de OCaml con varias banderas para proporcionar una mejor experiencia para los usuarios de NGP.

En general, que para ver dos tipos de construir artefactos: Los archivos JS generados y los metadatos que dependen de su OCaml confiaan en.

Ya que Commonjs no tiene espacios de nombre, para permitir que los archivos Js vivan en diferentes directorios, nosptros tenemos una bandera

bsc.exe-bs-paquete-nombre$npm_package_name-bs-paquete de salida modulessystem:path/path/a/su/jr/dir-ca.ml

Pasando esta bandera,bsc.exealmacenara sunombre-paquetey ruta de acceso relativa paquete.jsonen.cmjarchivos. Tambien generara archivos JS en el directorio especificado. Usted puede, y se le anima a almacenar archivos de JavaScript en un directorio jerarquico.

Para los artefactos binario ( Nota que esto no es necesario si solo desea que sus bibliotecas para ser consumido por los desarrolladores de JS, y tiene ventaja ya que los usuarios finales no nesecitan mas estos datos binarios),la convencion es almacenar todos los datos de.cmdatos ensolodirectoriopaquete.Json/lib/ocalmy javasscript archivos en unjerarquicodirectorio comopaquete.json/lib/js.

Use la biblioteca de OCaml como un paquete NPM

Si usted sigue la Convencion de diseño anterior, usando un paquete de OCaml es bastante sencillo:

bsc.exe-I ruta/a/ocaml/paqueta/instalado -c a.ml

Juntos

Su linea de comando seria como este:

bsc.exe -I ruta/a/ocaml/paquete 1/instalado -I ruta/a/ocaml/paquete 2/instalado bs-paquete-nombre $npm_package_name-bspaquete de salida commonjs:ruta/a/lib/js -ca.ml
