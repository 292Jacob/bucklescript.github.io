Intro to External

互操作系统的很多部分都会使用一个概念叫做external，接下来特殊介绍一下。

external 是定义BuckleScript或者OCaml或者Reason中的值的一个关键字

external myCFunction : int -> string = "theCFunctionName"

Reason语法：

external myCFunction : int => string = "theCFunctionName";

与JS中的let比较相似, 但external变量的主体是字符串， 上面的例子也是如此。 这个字符串通常会根据上下文会有一个特殊的意义， 在原生的Ocaml中，通常指的是C函数名称。 在BuckleScript里，external通常会与特定的[@bs.blabla]这样的属性一起使用。

一旦定义之后，你可以把external当做普通值来使用。

BuckleScript中的 external会被转换为JS值， 在编译时会被嵌入到调用函数里，随后会全部的删除和释放， 在js里不会看到有任何的输出。 就像是你自己手写的代码一样， 自然也不会有性能上的损失。

注意：不要在interface文件里使用external和[@bs.blabla]属性，否则，编译时不会进行嵌入。

外部特殊标识 Special Identity External

像下面这种external，值得特殊提一下：

external myShadyConversion : foo -> bar = "%identity"

Reason语法：

external myShadyConversion : foo => bar = "%identity";

This is a final escape hatch which does nothing but convert from the type foo to bar. 在后面的章节里，如果你无法编写出external代码，你可以暂时使用这种方案。 但最好不要这么写。
